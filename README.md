# マインドエンジニアリング・コーチング管理システム（Next.js版）

マインドエンジニアリング・コーチング業務管理のためのウェブアプリケーションです。Next.jsとGoogle Sheets APIを使用し、クライアント管理、セッション予約、支払い記録などの機能を提供します。

## 開発規約

### API設計規則

#### レスポンス形式の統一
すべてのリスト取得APIは以下の形式のJSONを返すこと：

```javascript
{
  "エンティティ名": [配列],  // 例：clients, sessions, payments
  "メタデータ": {}           // オプション：総件数、ページネーション情報など
}
```

例：
```javascript
// クライアント一覧
{
  "clients": [
    { "クライアントID": "C001", ... },
    { "クライアントID": "C002", ... }
  ]
}

// 支払い一覧
{
  "payments": [
    { "支払いID": "P001", ... },
    { "支払いID": "P002", ... }
  ]
}
```

#### APIユーティリティ関数
- データ取得関数は常にレスポンスを適切に処理し、期待される形式の配列を返すこと
- エラーハンドリングを適切に行い、エラーメッセージを明確にすること
- APIレスポンスの構造に依存するコードは、ユーティリティ関数内に閉じ込めること

### データモデルとステータス値の規則

#### ステータス値
すべてのステータス値は`lib/constants.js`で一元管理し、以下のルールに従うこと：

1. クライアントステータス
   - 問い合わせ（「問合せ」は使用しない）
   - トライアル前（「トライアル予約済」は使用しない）
   - トライアル済
   - 契約中（「継続中」は使用しない）
   - 完了
   - 中断

2. 支払いステータス
   - 入金済（「入金済み」は使用しない）
   - 未入金（「未払い」は使用しない）
   - キャンセル

3. セッションステータス
   - 予定
   - 実施済
   - キャンセル
   - 延期

#### 互換性の維持
- 古いデータ形式との互換性を確保するために、正規化処理を入れること
- 互換性のための変換は、データ取得後の処理で行うこと
- ステータスの表示と保存では、常に`constants.js`の定数を使用すること

### コンポーネント設計

#### データフローの原則
- データの流れは常にトップダウン（親から子へ）とすること
- APIデータの取得は最上位コンポーネントのみで行うこと
- 子コンポーネントへのデータ受け渡しは明示的なpropsを使用すること

#### エラーハンドリング
- データ取得時のエラーは常にキャッチし、ユーザーに適切なフィードバックを表示すること
- ローディング状態を明示的に管理し、ユーザーに読み込み中であることを伝えること
- データ不在時の代替表示を用意すること

## 環境のセットアップ

### 前提条件

- Node.js 14.x 以上
- npm または yarn
- Google Cloud Platformのアカウント

### インストール手順

1. リポジトリをクローン
```bash
git clone https://github.com/Yuuta12-maker/mec-coaching-nextjs.git
cd mec-coaching-nextjs
```

2. 依存パッケージをインストール
```bash
npm install
# または
yarn install
```

3. 環境変数の設定
`.env.example` ファイルを `.env.local` としてコピーし、必要な環境変数を設定します。

```bash
cp .env.example .env.local
```

4. Google Cloud Platformの設定
   - プロジェクトを作成
   - 必要なAPIを有効化（Google Sheets API、Google Calendar API、Gmail API）
   - OAuthクライアントの設定
   - サービスアカウントの作成とキーのダウンロード

5. 開発サーバーの起動
```bash
npm run dev
# または
yarn dev
```

6. ブラウザでアクセス: [http://localhost:3000](http://localhost:3000)

## デプロイ

Vercelへのデプロイは以下の手順で行います：

1. [Vercel](https://vercel.com) でアカウント作成（またはログイン）
2. 「New Project」をクリック
3. GitHub連携を設定し、このリポジトリを選択
4. 環境変数を設定
5. 「Deploy」をクリック
